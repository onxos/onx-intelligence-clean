## What Will Happen

### Service Selection Logic
- Both services are analyzed
- Scoring system evaluates:
  - Repository correctness (40 pts)
  - Branch correctness (30 pts)
  - Service status (20 pts)
  - Other factors (10 pts)
- Highest-scoring service becomes official

### Configuration Applied
The selected service will be configured with:
```yaml
Repository: onxos/onx-intelligence-clean
Branch: main
Runtime: Docker
Dockerfile: ./Dockerfile

Environment Variables:
  NODE_ENV=production
  PORT=10000
  JWT_EXPIRATION=24h
  DATABASE_URL=[from Render PostgreSQL]
  JWT_SECRET=[auto-generated by Render]

Health Check: /health
```

### Database
- Uses existing Render PostgreSQL database (intelligence-related)
- Connection string passed to service via DATABASE_URL
- No data deletion

## Troubleshooting

### If workflow fails:
1. Check workflow logs for error messages
2. Common issues:
   - Render API authentication error → verify RENDER_API_KEY secret
   - Service not found → verify service exists on Render
   - Deployment stuck → check Render dashboard for build issues

### If health check times out:
- This is normal during deployment
- Render takes 2-5 minutes to start service
- Check https://dashboard.render.com for build progress
- Retry manually after build completes

### If API docs not reachable:
- Service may still be starting
- Wait 1-2 minutes and try again
- Ensure /api/docs endpoint exists (it should from NestJS)
