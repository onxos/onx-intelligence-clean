# دستور السرعة — ONX SPEED CONSTITUTION v1.0
**التاريخ:** 2026-07-09 | **الحالة:** ملزم (L1 — تحت المبادئ السبعة مباشرة) | **المرجع:** MED v3.0 §4.3 (المكدس المقفول)

> **مبدأ المؤسس الحاكم:** «نظام مع كل ضخامته وقوته وقدراته أسرع وأخف من أسرع نظام ضخم
> بالعالم، وأسرع من أصغر نظام من صفحة واحدة.»
>
> هذا الدستور يحوّل المبدأ إلى **ميزانيات مقاسة ملزمة** — أي تجاوز لها عيب دستوري يُسجَّل
> ويُعالج كأولوية، بنفس صرامة معادلة OCMBR (MED §7.1).

## المادة 1 — الميزانيات الملزمة (Performance Budgets)

| المؤشر | الميزانية | طريقة القياس |
|---|---|---|
| حساب الخادم لكل طلب API (p95) | **< 100ms** | ttfb − tls من نقطة قريبة للخادم، أو server-timing |
| وصول من السعودية (TTFB p95) | **< 150ms** | curl من جهاز المؤسس |
| فتح أي شاشة (LCP) | **< 1s** | فحص متصفح |
| حزمة JS لأي شاشة | **< 250KB** مضغوطة | next build output |
| أي استعلام قاعدة بيانات (p95) | **< 20ms** | pg_stat_statements |
| نمو البيانات لا يغيّر الميزانيات | مليون سجل = نفس الأرقام | اختبار حمل دوري |

**قاعدة القياس قبل الادعاء:** لا يُعلن أي تحسين سرعة إلا برقم قبل/بعد قابل لإعادة القياس.

## المادة 2 — القرارات المعمارية المقفولة (لماذا نحن أسرع بنيةً)

1. **Monolith معياري لا microservices** — قفزة شبكية واحدة لكل طلب. أنظمة SAP/Oracle تدفع
   ضريبة 5-15 قفزة داخلية لكل شاشة؛ نحن قفزة واحدة. هذا القرار **مقفول** ولا يُفتح إلا بـCCR-A.
2. **الضخامة في القاعدة لا في الطلب** — 716 جدولاً و195 نظاماً لا تعني شيئاً لزمن الطلب الواحد
   إذا حمّلت كل شاشة شريحتها فقط (code-splitting عبر App Router — قائم) وسأل كل استعلام
   جدوله المفهرس فقط.
3. **الأحداث خلفية لا أمامية** — حين يُبنى الجهاز العصبي (المرحلة أ)، النشر في outbox
   لا يجوز أن يضيف > 5ms للطلب الأمامي؛ التسليم دوماً غير متزامن.

## المادة 5 — قواعد الكتابة الملزمة لكل كود قادم

1. كل `getAll` يُقيَّد بـ limit (سقف 100) — قائم في المصنع، يبقى قانوناً.
2. لا `SELECT *` في المسارات الساخنة حين تكبر الجداول — أعمدة القائمة فقط.
3. كل عمود يظهر في WHERE أو ORDER BY لجدول أعمال ⇒ مفهرس.
4. كل mutation تنشر حدثاً ⇒ النشر إدراج واحد غير متزامن (< 5ms).
5. أي صفحة جديدة تُقاس قبل الدمج: حجم حزمتها + LCP.
